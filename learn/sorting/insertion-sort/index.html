    • Insertion sort

    What?

    Stable sort with simple/compact code which sorts incrementally.

    Complexity

    • Time best: O(N) already/nearly sorted
    • Time worst: O(N^2) sorted backwards
    • Time average:
    • Space: O(1)

    Pros/cons

    • Pros:good for small N, stable sort, doesn't use recursion, no additional space used
    • Cons:slow for large N
